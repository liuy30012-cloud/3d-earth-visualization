# 3D 地球项目计划

## 项目概述

从零创建一个前后端分离的交互式 3D 地球项目，用于学习练手。前端以 React 框架为主，后端以 Spring Boot 框架为主。

## 技术栈

- **构建工具**：Vite（前端） / Maven（后端）
- **前端框架**：React 18 + TypeScript
- **3D 引擎**：Three.js（通过 @react-three/fiber + @react-three/drei）
- **后端框架**：Spring Boot 3 + Java 17+（或 Kotlin）
- **数据库**：MySQL 8（存储用户数据和可视化数据点）
- **认证**：JWT + 手机号登录/注册 + 邮箱登录/注册
- **前后端分离**：前端独立运行在 Vite 开发服务器，后端通过 REST API 提供接口，CORS 跨域

## 功能需求

### 1. 基础展示

- 3D 地球仪支持鼠标拖拽旋转、滚轮缩放、自动旋转
- 使用 NASA 蓝色大理石（Blue Marble）或类似高清地球纹理
- 支持光照和阴影效果，地球有明暗面

### 2. 完整地球

- 加载 GeoJSON 国家边界数据，在 3D 球面上绘制国界线
- 使用地形高度贴图实现地形起伏

### 3. 数据可视化

- **打点**：标记世界上所有城市（使用开源城市经纬度数据集，点击弹出信息卡片显示城市名称、坐标、国家等）
- **飞线**：城市之间绘制弧形飞线动画
- **热力图**：基于经纬度数据在球面上生成热力分布

### 4. 交互

- 点击城市标记弹出信息卡片（城市名称、坐标、国家等）
- 拖拽地球平滑旋转，惯性减速
- 侧边栏或控制面板切换不同可视化图层（打点/飞线/热力图）

### 5. 用户系统

- 登录（手机号 / 邮箱）/ 注册 / 退出
- JWT token 认证
- 手机号验证（短信验证码，可先用模拟验证码接口）
- 邮箱验证（发送验证链接或验证码）
- 用户个人信息（头像、昵称、邮箱）
- 用户可创建、保存、编辑自己的可视化图层配置

### 6. 后端 API

- 前端可独立启动（Vite 开发服务器）
- 后端独立启动（Spring Boot 内嵌 Tomcat）
- 前后端通过 REST API 通信，CORS 跨域
- 数据点存储：用户保存的城市标记、飞线、热力图数据持久化到数据库
- 数据接口支持按需加载（前端请求时返回对应 GeoJSON/坐标数据）

### 7. 自定义 Shader

- 大气层光晕效果（自定义 Shader 实现边缘发光）
- 飞线流光动画（Shader 控制颜色渐变和流动方向）
- 热力图渐变着色（自定义 Shader 实现平滑颜色过渡）

### 8. 响应式适配

- 桌面端（1920px+）完整功能
- 平板端（768px-1024px）自适应布局
- 移动端（< 768px）简化侧边栏为抽屉式，触控手势支持

## 项目结构要求

### 前端

- 模块化拆分：地球组件、标记组件、飞线组件、热力图组件各自独立
- 自定义 hook 封装 Three.js 逻辑
- 类型定义完整，不要 any

### 后端

- Spring Boot 分层架构：Controller → Service → Repository → Entity
- RESTful API 路由：auth（手机号/邮箱登录注册）、users、data-points
- Spring Security + JWT 认证
- MyBatis 或 Spring Data JPA 操作 MySQL
- 全局异常处理、CORS 配置
- 数据库迁移管理（Flyway 或 Liquibase）

## 工作流

1. `/plan` 先规划组件结构和坐标转换逻辑（经纬度 → 3D 球面坐标）
2. 搭建 Vite + React + TypeScript 前端 + Spring Boot 后端项目骨架
3. 先实现基础地球旋转缩放，验证纹理加载
4. 搭建后端 API（用户系统 + 数据点 CRUD）
5. 逐步叠加前端功能：国界 → 城市标记 → 飞线 → 热力图 → 自定义 Shader
6. 实现响应式适配
7. 每完成一个模块用 `/verify` 验证功能
8. 最后 `/code-review` 审查代码质量

## 验收标准

- 项目能 `npm start` 正常运行
- 地球可拖拽旋转、缩放
- 城市标记正确显示在球面上（经纬度映射准确）
- 飞线动画流畅
- 点击标记弹出信息
- 用户可注册、登录、保存可视化配置
- 移动端和桌面端均正常显示
- 大气层光晕、飞线流光等 Shader 效果可见
- 无 TypeScript 编译错误

## 改进与优化计划

> 以下从高频用户视角整理，按优先级分 P0（核心体验）/ P1（重要增强）/ P2（锦上添花）。

### P0 — 核心体验改进

1. **城市搜索功能**
   - 支持按城市名称模糊搜索
   - 搜索结果点击后自动旋转地球到该城市视角
   - 搜索框支持键盘导航（上下选择、回车确认）

2. **点击城市后平滑旋转视角**
   - 当前点击城市只弹出 InfoCard，没有视角变化
   - 需要相机平滑过渡（camera animation）到目标城市位置
   - 旋转过程中保留 OrbitControls 阻尼效果

3. **城市数据丰富化**
   - 扩展 City 接口：增加时区、GDP、海拔、城市描述、照片 URL
   - InfoCard 展示更多内容，布局美化（卡片式、分栏、图标）
   - 考虑接入第三方 API（天气、时区）获取实时数据

4. **热力图数据源明确化**
   - 当前热力图层没有明确的数据驱动逻辑
   - 应基于城市人口密度或用户自定义数据点生成
   - 球面热力图使用自定义 Shader 实现平滑渐变

5. **性能优化：飞线材质重建问题**
   - `Flyline` 组件在 `useMemo` 中依赖 `time`，导致每帧重建 ShaderMaterial
   - 改为通过 `useRef` + `useFrame` 更新 uniform，避免材质重建
   - 减少不必要的 GPU 内存分配和 GC 压力

6. **纹理本地缓存 / 降级方案**
   - 当前纹理依赖 `unpkg.com` 远程 CDN，网络不稳定时地球为黑色
   - 将纹理文件下载到 `public/` 目录作为本地资源
   - 加载失败时显示降级提示或占位图

### P1 — 重要功能增强

1. **城市筛选与分组**
   - 按大洲/国家/人口区间过滤城市
   - 图层面板增加筛选条件
   - 支持多选组合筛选

2. **自定义飞线连线**
   - 用户可自由选择两个城市生成飞线
   - 飞线可保存到用户配置（后端持久化）
   - 支持批量生成（选中多个起点和终点）

3. **加载状态指示**
   - 纹理/资源远程加载时显示 loading spinner
   - 加载完成后自动淡出
   - 加载失败时显示重试按钮

4. **用户操作指引**
   - 首次进入显示交互引导（拖拽旋转、点击城市、滚轮缩放）
   - 引导可关闭，状态存 localStorage
   - 支持重新查看（帮助按钮）

5. **快捷键支持**
   - `R` — 重置视角
   - `Space` — 暂停/恢复自转
   - `F` — 全屏切换
   - `1/2/3/4` — 快速切换图层

6. **主题/皮肤切换**
   - 提供多种地球纹理：卫星图、暗色主题、地形图
   - 大气层颜色可配置
   - 主题偏好存 localStorage

7. **截图/导出功能**
   - 将当前地球视角导出为 PNG 图片
   - 支持下载和复制到剪贴板
   - 导出时附带当前视角参数（经纬度、缩放）

8. **城市标记动态密度控制**
   - 根据相机距离动态调整城市标记显示密度
   - 远距离只显示主要城市，拉近后显示更多
   - 避免标记重叠遮挡

9. **用户配置持久化**
   - 图层开关状态、视角偏好、主题选择存 localStorage
   - 登录后同步到后端数据库
   - 下次打开时恢复上次状态

### P2 — 扩展功能

1. **3D 柱状图可视化**
   - 城市人口/GDP 数据用 3D 柱体从地表立起
   - 柱体高度映射数据值
   - 支持切换显示/隐藏

2. **昼夜分界线**
   - 基于实时时间计算太阳位置
   - 地球表面渲染明暗分界线
   - 夜间区域降低纹理亮度

3. **数据仪表盘**
   - 侧边面板展示全球统计数据
   - 城市数、总人口、各大洲分布饼图
   - 飞线连接统计

4. **多城市对比**
   - 选择两个城市并排对比信息
   - 双 InfoCard 模式
   - 显示两城市间飞线和距离

5. **时间轴**
   - 飞线动画关联历史时间线
   - 展示不同时期的航线/贸易路线变化
   - 时间轴播放/暂停控制

6. **分享链接**
   - 将当前视角/图层/主题状态编码到 URL 参数
   - 打开链接自动恢复到分享时的状态
   - 支持一键复制链接

7. **更多城市数据**
   - 补充非洲、南美、中东城市
   - 目标覆盖 100+ 城市
   - 支持从 CSV/JSON 文件导入

### 不要做

- 不做第三方 OAuth 登录（先做基础手机号+邮箱登录，后续可扩展）
